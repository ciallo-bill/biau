Release Checklist

公开 APK 前置条件

下载区保持关闭，直到每个公开发布条件都有可复现证据。当前只把 debug 构建作为内部验证信号，不把调试包、内部 artifact 或未审核候选包装成正式下载。

01

可复现构建

确认正式 release 构建命令、版本号和产物来源，避免使用临时 debug 包。

02

签名与校验

确定公开签名策略，并准备包体摘要，便于下载后校验。

03

基础回归

覆盖首页、孵化、社区、个人页和下载门禁，不把未验证路径开放给访客。

04

人审发布

确认生成候选、人审状态和包下载边界，不绕过人工视觉审核。

05

发布说明

准备版本说明、当前限制和后续计划，再把按钮替换为真实下载链接。

Download Gate

公开下载策略

当前页面不提供 APK 文件，也不放置占位下载链接。debug 构建只能证明 App 已进入内部验证，公开下载还需要完成正式 release 构建、签名策略、版本说明、基础回归和人工确认后再开启。

  • 已定位内部 debug 构建，可用于后续整理正式发布候选清单。
  • 不把 debug 包伪装成公开发布包。
  • 不绕过生成 - 人审 - 展示的状态门禁。
  • 不暴露内部 worker、私有 token、服务器地址或 artifact 路径。
  • 后续可接入正式 APK、版本号、校验摘要和更新日志。
