# 项目详情页图文案例化增强

## Goal

把现有项目详情页从“短介绍 + 单张顶部图”升级为访客可读的技术案例页：内容更完整，配图真实准确，并在正文中穿插 UI 截图、架构图、流程图、状态/质量证据图等多种视觉材料，让访客能快速理解每个项目做了什么、怎么实现、当前成熟度和后续方向。

## Requirements

- 项目详情页必须继续面向普通访客可读，不写成只给开发者看的内部文档。
- 每个重点项目需要有更完整的案例结构，至少覆盖：
  - 项目目标与使用场景。
  - 核心功能与体验入口。
  - 技术栈和关键实现。
  - 架构或数据流。
  - 当前上线/演示/发布状态。
  - 已知不足和后续优化方向。
- 配图必须与项目真实内容一致：
  - 不使用明显错误、无关或误导性的配图。
  - 优先使用真实站点截图、应用截图、已有展示图、实际 UI 状态图。
  - 无真实截图时，可以先使用明确标注为架构/流程/占位的 diagram，不伪装成真实产品截图。
- 正文需要支持穿插多个视觉块，而不是只有 hero 图：
  - UI 示例图。
  - 架构图。
  - 用户流程图。
  - 数据流/部署流图。
  - 状态检查或质量证据图。
- 视觉块需要具备标题、说明、类型和可选来源，方便以后扩展和维护。
- 页面在桌面和移动端都不能出现文字/图片重叠、卡片嵌套卡片、按钮文字溢出、布局跳动。
- 项目页内容必须与首页卡片、状态页、公开助手知识、外链可用性保持一致。
- 不提交私有后台截图、真实账号、密钥、内部控制台 URL、私有平台信息、未批准 APK 下载链接或敏感部署细节。

## Acceptance Criteria

- [ ] 审计当前项目详情页内容模型、页面渲染和图片资产，列出错误/薄弱图片与缺失图文位置。
- [ ] 设计并实现项目详情页正文视觉块数据模型。
- [ ] 至少为核心项目补充多段案例内容和正文视觉块。
- [ ] 替换或移除明显错误、无关、误导性的配图。
- [ ] 项目详情页在移动端和桌面端布局稳定，正文视觉块不会遮挡文字或按钮。
- [ ] 项目详情页内容与状态页、外链、下载门禁、公开助手知识不互相矛盾。
- [ ] 运行 `npm.cmd run lint`、`npm.cmd run build`、相关 UI/状态检查和 `git diff --check`。
