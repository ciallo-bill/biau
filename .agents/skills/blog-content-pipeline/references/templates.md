# Blog Content Templates

Use these as writing structures, not as mandatory headings. Keep public copy concrete, sanitized, and visitor-readable.

## 知识积累 / Knowledge Notes

- Title: concrete method or engineering problem.
- Detail: explain why this method matters and where it applies.
- Sections:
  - Problem boundary
  - Core mechanism
  - Engineering tradeoffs
  - Example from this site or a sanitized project
  - Common failure modes
  - Practical checklist
- Takeaways: 3 concise lessons that remain useful after the project context changes.

## 项目总结 / Project Notes

- Title: project + stage/result/tradeoff.
- Detail: summarize the stage and the key problem solved.
- Sections:
  - Project stage and goal
  - What changed or what was built
  - Architecture or workflow choice
  - Bugs, constraints, or tradeoffs
  - What the project still lacks
  - Next iteration direction
- Avoid repeating stable facts already on the project detail page.

## 资源分享 / Resource Picks

- Title: resource name or resource theme.
- Detail: explain why this resource is worth saving.
- Sections:
  - What it is
  - Why I recommend it
  - Best-fit scenarios
  - How I would use it
  - Limitations or caveats
  - Related alternatives
- Must include personal judgment; do not publish raw link collections.

## AI 日报 / AI Daily

- Title: date + focus topic.
- Detail: summarize the most useful signal from the day.
- Sections:
  - Today's highlights
  - What changed
  - Why it matters
  - What to try
  - Sources to revisit
  - Open questions
- Every news-like claim needs a source. If sources are weak, keep the item as a draft.

## 构建手记 / Build Log

- Title: system/workflow change + outcome.
- Detail: explain what was improved and why.
- Sections:
  - Starting point
  - Decision made
  - Implementation path
  - Verification
  - What became easier
  - Follow-up work
- Good for site evolution, Trellis workflow, assistant behavior, content governance, and release routines.

## Evidence Pack Checklist

- Source paths or URLs read.
- Facts safe to publish.
- Facts that are uncertain or stale.
- Details that must be omitted.
- Target column.
- Intended reader and value.
- Model used, if any.
- Review status.
